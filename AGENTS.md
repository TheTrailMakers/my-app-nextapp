# Tech Stack:

- Framework: Next.js 16
- Package Manager: bun
- State Management: Zustand
- UI Library: Tailwind CSS
- ORM: Drizzle ORM
- Database: PostgreSQL
- Authentication: better-auth
- API Routes: Next.js API Routes

# Things to keep in mind when running bash commands:

- Use bun, bunx instead of npm, npx.
- Assume bun dev is already running and app is available at http://localhost:3000.
- Any code changes should be validated by type checking and linting before finishing the task.

# Urls that doesn't require permission to access:

- nextjs.org
- tailwindcss.com
- better-auth.com
- orm.drizzle.team

# DONTS

- Don't delete useful code comments.
- Don't write unnecessary useEffect hooks.
- Look for reusable code and try not to repeat yourself.
- Don't create clever abstractions that make the code harder to understand.
- Don't hardcode colors, use theme colors setup in globals.css
- Don't hardcode fonts, use theme fonts setup in globals.css
